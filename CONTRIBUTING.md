# Contributing Guidelines

Спасибо за интерес к проекту! Этот документ поможет тебе внести свой вклад.

## Development Workflow

### 1. Создание ветки

```bash
# Обнови main
git checkout main
git pull origin main

# Создай ветку для задачи
git checkout -b feature/[название-задачи]
# или
git checkout -b bugfix/[название-бага]
```

### 2. Разработка

- Следуй соглашениям из [.ai/conventions.md](.ai/conventions.md)
- Пиши чистый, читаемый код
- Добавляй комментарии где необходимо
- Пиши тесты для новой функциональности

### 3. Тестирование

```bash
# Запусти все тесты
[команда тестов]

# Проверь coverage
[команда coverage]

# Запусти линтер
[команда линтера]

# Форматирование
[команда форматера]
```

### 4. Коммит

Используй понятные commit messages:

```
[type]: [краткое описание]

[опциональное подробное описание]
```

**Типы:**
- `feat:` - новая функциональность
- `fix:` - исправление бага
- `docs:` - изменения в документации
- `refactor:` - рефакторинг кода
- `test:` - добавление тестов
- `chore:` - рутинные задачи (обновление зависимостей и т.д.)

**Примеры:**
```bash
git commit -m "feat: add user authentication endpoint"
git commit -m "fix: resolve database connection timeout"
git commit -m "docs: update API documentation"
```

### 5. Push и Pull Request

```bash
# Push в свою ветку
git push origin feature/[название-задачи]

# Создай Pull Request в GitLab/GitHub
```

**В описании PR укажи:**
- Что сделано
- Почему (ссылка на issue или описание проблемы)
- Как протестировать

## Code Quality Standards

### Обязательно

- [ ] Код следует conventions из [.ai/conventions.md](.ai/conventions.md)
- [ ] Все тесты проходят
- [ ] Новая функциональность покрыта тестами
- [ ] Линтер не показывает ошибок
- [ ] Код отформатирован

### Желательно

- [ ] Обновлена документация (если нужно)
- [ ] Добавлены комментарии для сложной логики
- [ ] Проверена производительность (для критичных изменений)

## Типичные задачи

См. подробности в [.ai/common-tasks.md](.ai/common-tasks.md)

### Добавление новой функциональности

1. Создай issue или найди в backlog
2. Создай ветку `feature/[название]`
3. Разработай функциональность
4. Добавь тесты
5. Обнови документацию
6. Создай PR

### Исправление бага

1. Создай issue с описанием бага (или найди существующий)
2. Создай ветку `bugfix/[название]`
3. Добавь тест, воспроизводящий баг (если возможно)
4. Исправь баг
5. Проверь, что тест теперь проходит
6. Создай PR

### Рефакторинг

1. Убедись, что есть тесты для рефакторимого кода
2. Создай ветку `refactor/[название]`
3. Проведи рефакторинг
4. Проверь, что все тесты проходят
5. Создай PR с объяснением зачем

## Review Process

### Для автора PR

- Опиши что и зачем сделано
- Отметь reviewer'ов
- Будь готов к обсуждению и правкам

### Для reviewer'ов

- Проверь соответствие conventions
- Проверь наличие тестов
- Проверь читаемость кода
- Дай конструктивный фидбек

## Questions?

- Спроси в [канал/чат команды]
- Создай issue с вопросом
